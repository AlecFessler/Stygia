# Why we skip rather than delete:
#   * arch/aarch64/acpi.zig, arch/x64/acpi.zig         — ACPI table layout fields.
#     We model the full struct so reads at fixed offsets are correct even when
#     today's code only consumes a subset. Deleting fields would corrupt the
#     binary layout.
#   * arch/aarch64/kvm/regs.zig                        — EL2 sysreg bit-field
#     reference set (HCR_EL2 / VTCR_EL2 / VTTBR_EL2 / ESR_EL2 / CPTR_EL2 /
#     HPFAR_EL2 etc.). Kept whole so future EL2 bring-up grabs bits without
#     re-deriving from the ARM ARM.
#   * arch/x64/intel/vmx.zig, arch/x64/amd/svm.zig,
#     arch/x64/amd/vi.zig, arch/x64/intel/pmu.zig      — Intel/AMD MSR + VMCS /
#     VMCB / IOMMU / PMU encoding sets. Same reasoning: arch reference set,
#     kept whole.
#   * arch/x64/cpu.zig                                 — CPUID feature bits.
#     A future feature probe should pick the bit from this enum, not redefine
#     it; keep the spec-mandated value space.
#   * arch/x64/apic.zig                                — LAPIC + x2APIC MSR
#     register space. Hardware-defined enum range.
#   * arch/x64/gdt.zig (Tss), arch/x64/idt.zig         — x86 hardware-mandated
#     descriptor layout; reserved fields and IST slots stay so the binary
#     layout matches the CPU's expectations.
#   * arch/x64/paging.zig, arch/aarch64/paging.zig     — page-table entry bits
#     (PageEntry). Hardware layout fidelity; flags like dirty/contiguous etc.
#     are read by the MMU even if the kernel never asserts them today.
#   * arch/aarch64/exceptions.zig (ExceptionClass),
#     arch/aarch64/kvm/regs.zig (EC),
#     arch/aarch64/gic.zig                             — ARM ESR_EL1 / ESR_EL2
#     ExceptionClass / GICD register enum value spaces.
#   * arch/aarch64/interrupts.zig, arch/aarch64/pmu.zig — register-block /
#     register-accessor parity. x18 is part of the AArch64 GPR file even when
#     the trap save path doesn't touch it; PMOVSCLR is a documented PMUv3
#     register accessor reserved for the overflow-ISR bring-up.
#   * arch/x64/serial.zig (com2..com4), arch/x64/exceptions.zig
#     (invalid_execute), arch/x64/timers.zig                 — hardware port /
#     register / vector identifiers. Kept for hardware fidelity.
#   * arch/aarch64/vm.zig                              — aarch64 VmPolicy ABI
